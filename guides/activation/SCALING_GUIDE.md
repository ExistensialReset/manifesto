## PREAMBLE: THE DUNBAR REALITY

**Dunbar’s Number (~150):**  
Humans can only maintain stable, trust-based relationships with a limited number of people.

Beyond this threshold, systems naturally:
- Fragment into subgroups, or
- Harden into hierarchies

**Flow’s central challenge:**  
How do we scale **without becoming what we are escaping**?

**Flow’s answer:**  
Nested autonomy.  
Fractals, not pyramids.

---

## §1. THE FOUR PHASES OF SCALING

Flow scales in **qualitatively different phases**, not linear growth.

### Phase 1: Pilot Circle (5–20 people)
**Focus:** Trust, proof, safety

### Phase 2: Local Node (20–100 people)
**Focus:** Stability, infrastructure, resilience

### Phase 3: Regional Network (100–500 people)
**Focus:** Coordination, specialization, shared reserves

### Phase 4: Bioregional Coordination (500+ people)
**Focus:** Ecology, long-term resilience, external interface

**Important:**  
Each phase has **different risks, protocols, and ethical priorities**.  
Scaling too early is more dangerous than not scaling at all.

---

## §2. PHASE 1 — PILOT CIRCLE (5–20 PEOPLE)

### 2.1 Core Intent
**Primary objective:**  
Demonstrate that Flow works *in lived reality*.

Focus on:
- Psychological safety
- Baseline security
- Trust formation
- Learning to resolve conflict without authority

### 2.2 Core Practices
- Weekly gatherings (check-in, coordination, conflict resolution)  
- Shared meals (3–5 per month)  
- Baseline secured for all members  
- One shared project (garden, workshop, learning space, etc.)

### 2.3 Indicators of Health
- Baseline stable for 6+ months  
- Reduced stress and anxiety (L)  
- Emergent, unplanned initiatives (S)  
- Conflicts resolved internally, without coercion (I)

### 2.4 Structure
**Rotating roles (3–6 month terms):**
- Coordinator (process holder, not leader)  
- Treasurer (Baseline + Critical Reserve)  
- Conflict Mediator  
- External Communicator (if needed)

**Non-negotiables:**
- No permanent roles  
- No charismatic ownership  
- No founder immunity

**Decision-making:** Consent-based (see PROTOCOL.md).

### 2.5 Common Failure Modes
- **Founder syndrome** → solve with explicit rotation  
- **Burnout** → slow down, redistribute, rest  
- **Mission drift** → regular “Why are we here?” check-ins

## §3. PHASE 2 — LOCAL NODE (20–100 PEOPLE)

### 3.1 When Scaling Is Allowed
Only scale when:
- Pilot stable for 12+ months  
- Baseline consistently protected  
- Conflicts resolved without escalation  
- New people are waiting *because it works*  

Otherwise: **do not scale.**

### 3.2 Structural Evolution
#### A) Sub-Circles (8–15 people)
- Divide into autonomous sub-circles  
- Each owns its domain  
- Coordination happens via delegates  
- Sub-circles meet weekly, full node meets monthly

#### B) Shared Infrastructure
- Housing, community kitchen, workshop/makerspace, refugium  
- Exists to **reduce pressure**, not increase productivity

#### C) Critical Reserve
- Target: ≥30% of monthly Baseline costs  
- Absorb shocks, prevent panic, preserve dignity

### 3.3 Decision-Making at Scale
Consent slows as size grows → **subsidiarity**  
Authority flows downward:
1. Individual  
2. Sub-circle  
3. Local node  
4. Regional network (later)  
Only escalate when necessary. Never centralize for convenience.

### 3.4 Communication Systems
- Async communication essential  
- Meetings pruned aggressively  
- Recommended: Channel-based chat, monthly assemblies, written summaries

### 3.5 Failure Risks
- Delegate power ossification  
- Sub-circle isolation  
- Process bloat  

**Corrective action:** Pause → Simplify → Rotate → Rest

## §4. PHASE 3 — REGIONAL NETWORK (100–500 PEOPLE)

### 4.1 Entry Conditions
- Multiple healthy local nodes  
- Informal resource exchange happening  
- Desire for coordinated resilience

### 4.2 Inter-Node Council
- 2 rotating delegates per node, 6-month terms  
- Consent-based proposals only  
- No enforcement, override, or unilateral allocation

### 4.3 Regional Critical Reserve
- ~30% of total regional Baseline needs  
- For shocks beyond single-node capacity  
- Loans, not domination

### 4.4 Specialization Without Dependency
- Nodes may specialize (food, healthcare, housing, learning, manufacturing)  
- **Rule:** No node becomes indispensable

---

## §5. PHASE 4 — BIOREGIONAL COORDINATION (500+ PEOPLE)

### 5.1 Preconditions
- Long-term stability, ecological awareness, multi-node maturity  
- Optional, rare

### 5.2 Bioregional Assembly
- Focus: water, land, energy, ecological balance  
- Authority: propositional only, no enforcement

### 5.3 Ecology Without Authoritarianism
**Anti-eco-fascism principle:** Human dignity is not sacrificed for ecological purity

## §6. PRESERVING FLOW AT SCALE

**Core Tensions**
- Efficiency vs autonomy → autonomy wins  
- Intimacy vs scale → nesting resolves  
- Stability vs experimentation → dual-memory

**Anti-Hierarchy Safeguards**
- Rotation, term limits, recall rights, radical transparency, right of exit  
- If hierarchy persists → Split. Healthy division is success

## §7. HEALTH METRICS

### Quantitative
- Baseline security (target: 100%)  
- Reserve levels (≥30%)  
- Participation (≥70%)  
- Conflict resolution speed (<30 days)

### Qualitative
- Calm  
- Trust  
- Mutual aid  
- Emergent creativity
