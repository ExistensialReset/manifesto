# LOTUS AMENDMENT CHECKLIST
## Quick Reference for Baseline Amendment Proposals

**Reference:** `BASELINE_AMENDMENT_PROTOCOL.md` v1.1  
**Purpose:** Ensure all amendments follow constitutional rules, audit requirements, and risk triggers.

---

## 1. Proposal Basics
- [ ] Title and Proponent defined  
- [ ] Metrics affected identified (`RESOURCE_METRIC_STANDARDS.md`)  
- [ ] Current vs Proposed values recorded  
- [ ] Effective date and phased timeline included  
- [ ] Regional feasibility assessed  

---

## 2. Amendment Type
- [ ] Increase → ≥66% LOTUS approval, 30-day public review, AI simulation report  
- [ ] Decrease → ≥75% LOTUS approval, 60-day review, dual AI simulation, regional feasibility audit  
- [ ] Refinement → Metric clarification only, no numeric change  

---

## 3. Emergency / Temporary Reduction
- [ ] Trigger condition documented (war, planetary, systemic)  
- [ ] ≥75% LOTUS vote  
- [ ] Automatic expiration (90 days) noted  
- [ ] Post-facto ratification report due within 30 days  

---

## 4. Monitoring & Audit
- [ ] Monitoring plan defined (1/3/6/12 months + annual)  
- [ ] KPIs linked to thresholds (`RISK_MANAGEMENT.md`)  
- [ ] Responsible owners listed  
- [ ] Recovery protocol prepared (`BASELINE_RECOVERY_PROTOCOL.md`)  

---

## 5. Revision Lock & Exceptions
- [ ] Lock period (12 months) noted  
- [ ] Emergency override specified (if applicable)  
- [ ] Rare non-emergency override requires ≥90% LOTUS + independent audit  

---

## 6. Recordkeeping
- [ ] Proposal recorded in `/compostandgrowth/baseline_amendments/`  
- [ ] Immutable timestamp & hash included  
- [ ] Ledger entries complete (proposal, vote, AI reports, monitoring)
